# Database Schema - Complete Reference

An explanation of the current PostgreSQL structure used by the Product Store app.

---

## Database Overview

The database is named `dotnet_db` and currently has one table: `Products`.

The backend code uses DTOs and a service layer, but the database still stores one simple entity:

```
React JSON
  -> Product DTO
  -> ProductService
  -> Product entity
  -> Products table
```

---

## Products Table

The `Products` table stores product records.

### EF Core Migration Shape

The current migration creates the table like this:

```sql
CREATE TABLE "Products" (
    "Id" integer GENERATED BY DEFAULT AS IDENTITY,
    "Name" text NOT NULL,
    "Description" text NOT NULL,
    "Price" numeric NOT NULL,
    CONSTRAINT "PK_Products" PRIMARY KEY ("Id")
);
```

### Columns Explained

| Column | Type | Required | Description |
|--------|------|----------|-------------|
| `Id` | integer identity | Auto | Unique identifier generated by PostgreSQL |
| `Name` | text | Yes | Product name |
| `Description` | text | Yes | Product details |
| `Price` | numeric | Yes | Product price stored as a decimal |

## Model and DTO Mapping

### Database Entity

`Models/Product.cs` represents the database row:

```csharp
public class Product
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string Description { get; set; } = string.Empty;
    public decimal Price { get; set; }
}
```

### API DTOs

The API does not accept raw `Product` entities from the client.

| DTO | Used By | Fields |
|-----|---------|--------|
| `CreateProductDto` | `POST /api/products` | `Name`, `Description`, `Price` |
| `UpdateProductDto` | `PUT /api/products/{id}` | `Name`, `Description`, `Price` |
| `ProductResponseDto` | API responses | `Id`, `Name`, `Description`, `Price` |

The `Id` is not sent in create/update request bodies. It is generated by the database and returned in responses.

## Common SQL Queries

### Get All Products

```sql
SELECT * FROM "Products";
```

### Get One Product

```sql
SELECT * FROM "Products" WHERE "Id" = 1;
```

### Search by Name

```sql
SELECT * FROM "Products" WHERE "Name" ILIKE '%keyboard%';
```

### Products Under $100

```sql
SELECT * FROM "Products" WHERE "Price" < 100;
```

### Count Products

```sql
SELECT COUNT(*) FROM "Products";
```

### Average Price

```sql
SELECT AVG("Price") FROM "Products";
```

## Changing Data

### Add a Product

```sql
INSERT INTO "Products" ("Name", "Description", "Price")
VALUES ('Monitor', '27-inch 4K display', 399.99);
```

### Update a Product

```sql
UPDATE "Products"
SET "Name" = 'Gaming Monitor', "Price" = 449.99
WHERE "Id" = 4;
```

### Delete a Product

```sql
DELETE FROM "Products" WHERE "Id" = 4;
```

## SQL Operations Mapped to API

| Operation | SQL | API Endpoint | Method |
|-----------|-----|--------------|--------|
| Read All | `SELECT *` | `/api/products` | GET |
| Read One | `SELECT * WHERE "Id" = 1` | `/api/products/1` | GET |
| Create | `INSERT INTO` | `/api/products` | POST |
| Update | `UPDATE ... WHERE "Id" = 1` | `/api/products/1` | PUT |
| Delete | `DELETE WHERE "Id" = 1` | `/api/products/1` | DELETE |

## Migrations

Create and apply migrations from the backend project folder:

```bash
cd backend/DotNetApi
dotnet ef migrations add MigrationName
dotnet ef database update
```

The existing initial migration creates only the `Products` table. If you add a new property to `Product`, create a new migration and apply it with `dotnet ef database update`.

## Backup and Recovery

### Backup the Products Table

```bash
pg_dump -U postgres -d dotnet_db -t "Products" > products_backup.sql
```

### Backup the Whole Database

```bash
pg_dump -U postgres -d dotnet_db > full_backup.sql
```

### Restore

```bash
psql -U postgres -d dotnet_db < full_backup.sql
```

## Key Concepts

| Term | Meaning |
|------|---------|
| **Table** | Like a spreadsheet with rows and columns |
| **Column** | A property/field such as `Name` or `Price` |
| **Row** | One complete product record |
| **Primary Key** | Unique identifier for each row |
| **DTO** | API request/response shape |
| **Entity** | C# class that maps to the database table |
| **Migration** | EF Core instructions for changing the database |
